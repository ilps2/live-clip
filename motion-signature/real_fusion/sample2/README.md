# V3：语义合并 + 成品切片（final_cut_sample2.mp4）

## 1. 语义合并

方法：对 7 个候选的 ASR 文本做 2–4 字关键词 Jaccard 相似度。
**结果：全部 ≈0**——ASR 错别字（"素颜霜"→"顺双"、"提亮"→"抓体"等音近误识）
使字面重叠失效。这本身是发现：跨候选相似度需要语义层（词向量/商品词表），
字面方法在方言带货 ASR 上不可用。

改用商品特征词人工核验（读原文），结论：
- **cand4+cand5+cand6+cand7 = 同一"素颜霜"讲品大周期（455.8–705s）**——
  用户确认的 4&5、6&7 两对实为同一周期的一段：效果演示→价格机制→逼单→
  上手对比→指尖试涂→拆用试用→防水防汗→拍三支收尾
- cand1（89–164s）= 防晒下水测试演示，独立周期
- cand2/cand3 = 人群适配/转场片段

## 2. 选品与语义边界

选中 **cand4+cand5 合并事件**（逼单高潮，用户已确认高光正确），压缩到 ≤60s：

- **起点 461.84s**："这款虽然它是没有什么遮瑕效果的，说是把你提亮肤色，
  让你（像）妈生的好皮肤"——产品定位的自然开头（前一句 458–461 在聊
  粉底液/遮瑕膏对比，属于铺垫）
- **终点 520.08s**："放心拍就可以了。今天赶上姐妹们"——促单收尾话术结束
  （521s 起转入"皮肤黑的……"新一轮人群拓展，属下一周期）
- 切片 461.84–520.08 = **58.27s**，内容弧：提亮效果展示 → 领 5 元券 →
  3 支 164（均价 50）→ 售后保障（运险/处理到满意）→ 稀缺催促
  （"我就要下播了""真的要等双十一了"）→ "放心拍"收尾

## 3. 首尾话术原文（验证完整性）

开头两句：
- 461.84s「这款虽然它是没有什么遮瑕效果的」（产品引入，非半句）
- 464.10s「说是把你抓体量肤色」（"提亮肤色"误识）

结尾两句：
- 517.98s「放心拍就可以了」
- 519.08s「今天赶上姐妹们」（催促收尾完整，下一句 521.08 已属新周期）

## 4. 剪辑与音频修复验证

旧问题：`-c copy` 流拷贝从最近关键帧开始 + 时间戳问题导致播放器开头无声。
修复：`-ss`/`-to` 放 `-i` 之后（精确 seek）+ 视频 libx264 重编码
（首帧即关键帧）+ 音频 AAC 128k 重编码。

验证（ffprobe + volumedetect）：
- 时长 **58.27s ≤ 60s** ✓，视频 h264 + 音频 aac 双流 ✓
- 全片音量 mean **-20.6 dB** / max -0.5 dB；前 3 秒 mean **-21.5 dB** /
  max -4.8 dB —— 开头即有正常语音电平 ✓

产出：`real_fusion/sample2/final_cut_sample2.mp4`

# 实验：语义时间压缩剪辑（声画错位，compressed_cut_sample2.mp4）

在同一讲品大周期（455.8–705s ≈ 250s）内做"视觉轨=静音演示画面拼接、
音频轨=错位时间点逼单语音拼接"的高密度切片，与 V3 忠实切片对比。

## 选段 plan

### 视觉轨（静音硬拼，累计 44.97s）

| 序 | 原片时间 | 时长 | 内容 | 选择理由 |
|---|---|---|---|---|
| V1 | 467.06–470.56 | 3.5s | 上手效果展示（面部+双手入镜） | 周期内首个"效果"画面，开篇抓眼 |
| V2 | 527.50–540.00 | 12.5s | 手臂对比"脸白身上黄黑皮" | 强对比演示，面部入镜但有手势动作 |
| V3 | 568.64–597.64 | 29.0s | 指尖指腹试涂"直接化开不卡粉" | 周期内运动 z 最高段(+0.50)，且为手部特写、口型不可见，错位最安全 |

抽帧核验（468.5 / 534 / 592 / 700s）：592s 确认为纯手部特写无脸；
467/534/700s 面部清晰可见（口型冲突风险段，见下）。

### 音频轨（错位拼接，接缝 acrossfade 0.25s，累计 44.84s）

| 序 | 原片时间 | 时长 | 原文（首尾） | 信息点 |
|---|---|---|---|---|
| A1 | 473.70–490.46 | 16.8s | 「领优惠券…」→「拍3只的话给才一支」 | 5元券+3支164均价50 |
| A2 | 493.60–502.14 | 8.5s | 「用得好用…」→「有说过保障好不好」 | 售后运险保障 |
| A3 | 617.60–620.34 | 2.7s | 「这款顺双可以拆写/先拆一只写来试用」 | 拆用试用 |
| A4 | 685.54–689.48 | 3.9s | 「拍三支的话…可以拆一支症状才先试一下」 | 三支拆退规则 |
| A5 | 505.22–519.08 | 13.9s | 「我就要下播了」→「放心拍就可以了」 | 稀缺催促+放心拍收尾（V3 同款收尾） |

音频顺序为顺向销售逻辑（优惠→保障→试用规则→催促收尾），全部剪在
句子边界（词级时间戳核对），画面指示句（"你看/看到没"）不配错位语音。
注：本周期内 product 类运动帧几乎不触发（仅 1 帧），视觉段以"运动 z 峰 +
转录指示词 + 抽帧人工核验"联合确定。

## 已知风险（诚实记录）

1. **口型冲突（高）**：V1（467–470）与 V2（527–540）段主播正对镜头、
   口型清晰可见，配的是 473–490 的领券价格语音——懂行观众能察觉
   "嘴型对不上"。占全片约 35%（16/45s）。
2. **语气跳跃（低）**：音频按销售逻辑顺排，接缝在句号处且 0.25s
   交叉淡化，预期过渡自然；但我无法自查听感，仅能依据句边界与电平推断。
3. **语义违和（中低）**：V3 指尖试涂画面配 A2 售后/A3 拆用语音时，
   "画面在演示、声音在讲规则"，属于可接受的带货切片常见形态；
   未将黑布测试(633)/防水测试(700)画面配非测试类语音（规避了最强违和）。
4. **信息过载风险**：7 个信息点压进 45s，语速不变下密度 ~2.3倍于原片，
   可能显得"赶"。

## 预注册式成败预判

- **预测**：信息密度显著提升（目标达成，已验证：7 信息点/45s）；
  口型冲突是可感知穿帮，集中在 V1/V2 两段。
- **成败判定标准**：若观看者认为"像一条正常高密度带货切片、无突兀感"
  ≥大部分段落，则方向成立；否则视觉轨应改为以 B-roll 式特写段
  （如 568–597 指尖试涂这类口型不可见镜头）为主、面部段仅短点綴。
- **待人工确认**：成片听感/观感（我无听觉自查能力）。
